### Integration Issues

1. **Journal data gap (Medium):** `_record_to_journal_v2()` omits `direction` and `entry_ideal` from `journal_data`. Both columns always stored as NULL. `/api/chat` context shows "N/A" for direction.

2. **Implicit dict contract (Low):** `ta_data.key_levels` between TAMentor and RiskAgent has no shared Pydantic model. Silent degradation to percentage-based stops if return shape changes.

3. **tp2 assertion too strict (High for tests):** `test_integration_pipeline.py` line 125 asserts `tp2` must be non-null float for all non-Avoid signals. `RiskAgent` legitimately returns `tp2=None` when only one S/R target qualifies. Will fail on live data.
