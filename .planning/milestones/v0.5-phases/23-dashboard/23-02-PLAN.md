The backend NansenSignal model has 5 sub-signals: exchange_flows, fresh_wallets, smart_money, top_pnl, whale_activity.
The nansen_summary array typically has one string per signal type (up to 5 elements).
Map index 0→Smart Money Flows, 1→Exchange Flows, 2→On-Chain Holdings, 3→Buy/Sell Pressure, 4→Top PnL Wallets.
If the array is shorter, render only the available entries.
Parse bullish/bearish/neutral direction from the string content: if it contains "bullish" or "accumulating" or "outflow" → bullish; if "bearish" or "distributing" or "inflow" (exchange inflow = bearish for price) → bearish; else neutral.
Anomalous = bearish direction (border-amber-500/50).
