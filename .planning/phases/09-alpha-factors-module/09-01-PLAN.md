  <action>
Create `src/backend/analysis/alpha_factors.py` with four calculation functions following the established indicators.py pattern:

1. **calculate_momentum_score(df, short_period=10, long_period=20)**
   - Compute ROC: ((close[-1] - close[-n]) / close[-n]) * 100 for short (10) and long (20) periods
   - Combine: 60% short + 40% long
   - Normalize using np.tanh(combined / 10) * 100 to bound to -100/+100
   - Add np.clip safety for edge cases
   - Return dict: {'short_roc', 'long_roc', 'momentum_score'}
   - Minimum data: long_period + 1 (21 candles)

2. **detect_volume_anomaly(df, ma_period=20, threshold=2.0)**
   - Compute current volume and 20-period rolling mean
   - Calculate ratio = current / average
   - Flag is_anomaly = ratio > threshold (2.0)
   - Handle zero avg_volume edge case (return None)
   - Return dict: {'current_volume', 'avg_volume', 'volume_ratio', 'is_anomaly'}
   - Minimum data: ma_period (20 candles)

3. **calculate_ma_deviation(df)**
   - Compute EMAs using df['close'].ewm(span=X, adjust=False).mean()
   - Use adjust=False for TradingView-compatible calculation
   - Calculate % deviation: ((price - EMA) / EMA) * 100
   - Return dict: {'deviation_20', 'deviation_50', 'deviation_200'}
   - Minimum data: 200 candles

4. **calculate_volatility_score(df, atr_period=14)**
   - Import and call calculate_atr from indicators.py (DO NOT reimplement)
   - Normalize: atr_percent = (atr / current_price) * 100
   - Scale: volatility_score = min(atr_percent * 20, 100.0) (5% ATR = 100)
   - Return dict: {'atr', 'atr_percent', 'volatility_score'}
   - Minimum data: atr_period + 1 (15 candles)

Include proper docstrings with Args, Returns, Minimum data sections matching indicators.py style.
All functions return Optional[dict] - None for insufficient data.
