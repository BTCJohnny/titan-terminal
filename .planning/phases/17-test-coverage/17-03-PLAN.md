**Setup:** All tests must use in-memory SQLite by patching `get_signals_connection`. Create a pytest fixture:

```python
@pytest.fixture
def mock_db():
    """In-memory SQLite database for testing."""
    conn = sqlite3.connect(':memory:')
    conn.row_factory = sqlite3.Row
    with patch('src.backend.db.signals_db.get_signals_connection', return_value=conn):
        yield conn
    conn.close()
```

**IMPORTANT:** The functions in signals_db.py call `get_signals_connection()` internally and also call `conn.close()`. Since we want to inspect the database state after function calls, we need to handle this carefully. Override the `close` method on the test connection to be a no-op: `conn.close = lambda: None` inside the fixture, and close it for real in the fixture teardown.

Updated fixture pattern:
```python
@pytest.fixture
def mock_db():
    conn = sqlite3.connect(':memory:')
    conn.row_factory = sqlite3.Row
    real_close = conn.close
    conn.close = lambda: None  # Prevent signals_db functions from closing our test connection
    with patch('src.backend.db.signals_db.get_signals_connection', return_value=conn):
        yield conn
    real_close()  # Actually close when fixture tears down
```
