**analyze() method signature:**
```python
def analyze(
    self,
    symbol: str,
    entry_price: float,
    stop_price: float,
    trade_direction: Literal["long", "short"],
    sr_levels: dict,  # {'support': [float], 'resistance': [float]}
    open_position_count: int = 0,
    account_size: Optional[float] = None,
) -> RiskOutput:
```

**Implementation logic:**

1. **Derive stop from S/R levels**:
   - For long: stop = nearest support level BELOW entry_price. If proposed stop_price is tighter (closer to entry) than the S/R-derived stop, use the S/R-derived stop (wider is safer). If no support levels available, use the proposed stop_price.
   - For short: stop = nearest resistance level ABOVE entry_price. Same tighter-vs-wider logic.
   - Stop type = "structure" when S/R-derived, "percentage" when using proposed stop.

2. **Derive targets from S/R levels**:
   - For long: TP1 = nearest resistance level giving >= 3:1 R:R. TP2 = next resistance level beyond TP1. If no resistance levels meet 3:1, set tp1 to the best available and let R:R check flag it.
   - For short: TP1 = nearest support level giving >= 3:1 R:R. TP2 = next support level beyond TP1.
   - If S/R levels are empty, set TP1/TP2 to None and flag in rejection_reasons.

3. **Calculate R:R ratio**: `abs(tp_price - entry_price) / abs(entry_price - final_stop_price)`

4. **Calculate risk percent** (if account_size provided):
   - `risk_per_unit = abs(entry_price - final_stop_price)`
   - `position_size_units = (account_size * MAX_RISK_PER_TRADE) / risk_per_unit`
   - `suggested_position_percent = (position_size_units * entry_price / account_size) * 100`

5. **Enforce 3 Laws**:
   - Law 1 (2% risk): If account_size provided, verify risk_amount <= account_size * 0.02. Always "pass" if no account_size (can't check without it).
   - Law 2 (3:1 R:R): Check R:R to TP1 >= MIN_RISK_REWARD (3.0). "fail" if below.
   - Law 3 (max positions): Check open_position_count < MAX_POSITIONS. "fail" if >= MAX_POSITIONS.

6. **Set approved/rejected**:
   - `approved = True` only if ALL 3 Laws pass
   - Collect rejection_reasons for each failed law
   - If no valid TP1 found from S/R levels, add "No valid target from S/R levels" to rejection_reasons

7. **Build and return RiskOutput** with all fields populated. Use `trade_direction="avoid"` and `FinalVerdict(action="avoid", ...)` when rejected.
